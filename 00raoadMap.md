# Java From Scratch — Curriculum Roadmap (Milestone 1: Core Java → OOP)

This is the sequencing for the first milestone. Each numbered file in this folder corresponds to one GitHub Issue. Copy the content of each file into a new Issue using the repo's "Student Task" issue template — paste the body under **Task Objective**, **Requirements / Checklist**, and **Tutor Notes** fields respectively.

## Suggested order (do not skip ahead — each one builds on the last)

| # | Topic | Depends On | New Concept Introduced |
|---|---|---|---|
| 1 | Java Setup & First Program | — | JDK, compiling, running |
| 2 | Variables & Data Types | 1 | primitives, String, casting |
| 3 | Operators & Expressions | 2 | arithmetic, modulus, logical ops |
| 4 | Conditionals | 3 | if/else, switch, String comparison |
| 5 | Loops | 3, 4 | for, while, do-while, break/continue |
| 6 | Arrays | 5 | indexed storage, for-each |
| 7 | Methods | 2–6 | reusable logic, parameters, return values |
| 8 | OOP: Classes & Objects | 7 | blueprint vs instance, constructors |
| 9 | OOP: Encapsulation | 8 | private fields, validation, `this` |
| 10 | OOP: Inheritance | 8, 9 | extends, super, @Override |
| 11 | OOP: Polymorphism & Abstraction | 10 | abstract classes, interfaces |

## How to use these with the repo workflow

1. Open each `.md` file in this folder
2. Go to your repo → **Issues** → **New Issue** → choose **Student Task** template
3. Fill in:
   - **Assigned Student**: their GitHub username
   - **Difficulty / Tech Stack**: copy from the top of each file
   - **Task Objective**: copy the "Objective" section
   - **Requirements / Checklist**: copy the checklist items (GitHub renders `- [ ]` as clickable checkboxes automatically)
   - **Tutor Notes**: copy the "Tutor Notes" section
4. Assign it, and the student picks it up following the PR process already documented in your README

## After Milestone 1

Once the student has merged PRs for all 11 issues and can explain OOP's four pillars (encapsulation, inheritance, polymorphism, abstraction) in their own words, they're ready for **Milestone 2**: Collections, Exception Handling, and File I/O — just say the word when you're ready and I'll build that set the same way.
